CSS

is presentation

it is a web page's skin

CSS in a web page has

no required elements

think of CSS as

the style police

oh no you di'int!

CSS can be added to a web page either

internally or externally

(like attaching an image to an email)

The internet has been built on the idea of

separate structure (HTML) & style (CSS)

to improve the clarity of the web

<head>
    <title>Page Title</title>
    
    <link rel="stylesheet" type="text/css" href="styles.css">
    
</head>

Keeping with this design, we add external CSS through a

link element in the head element

On a side note,

CSS is only allowed in the head element

because browsers build the web page top down

If the CSS is last,

you'll see the skeleton before you see the skin

(this is not pleasant)

CSS rules are composed of three parts:

selector

a selector

{ }

curly braces

property:value;

a set of property definitions

A CSS selector tells us

which HTML element(s)

the rule should apply to

(one CSS rule can apply to many HTML elements depending on the selector)

A CSS selector can be

a tag, #id, .class, or combination

A tag selector selects

all HTML elements of that tag type

For example:

This rule with a tag selector

p {
    property:value;
}
<h1>...</h1>
<p><!--select--></p>
<p><!--select--></p>

will only apply to paragraph elements

An id is a property of a HTML element that gives it

a unique identity

An #id selector selects

all HTML elements with that id

(but an id should always be unique!)

For example:

This rule with an #id selector

#unique {
    property:value;
}
<h1>...</h1>
<p>...</p>
<p id="unique"><!--select--></p>

will only apply to the unique element

class 1

class 2

A class is a property of a HTML element that makes it

part of a group of elements

<div id="unique" class="class1 class2 etc">...</div>

While a HTML element can only have 1 id, it may have

as many classes as necessary

(each separated by a space)

A .class selector selects

all HTML elements with that class

(classes are not unique and should be reused often)

For example:

This rule with a .class selector

.common {
    property:value;
}
<div class="common"><!--select--></div>
<h1>...</h1>
<p class="common"><!--select--></p>

will apply to all elements of that class

In the last example, notice that

we used a .class selector to select

a division & a paragraph

But what if we wanted

only the paragraph?

(this is where combination selectors come into play)

A combination selector is

two selectors joined together

(each can be a combination selector itself!)

selectorAselectorB

Combination selectors joined without a space look for

everything in a single HTML element

For example:

This rule with a parent combination selector

div.common {
    property:value;
}
<div class="common"><!--select--></div>
<h1>...</h1>
<p class="common">...</p>

will apply to all divisions of class common

selectorA  selectorB

Combination selectors joined using spaces look for

a parent matching the left with a child matching the right

For example:

This rule with a parent-child combination selector

div .common {
    property:value;
}
<div>
    <p class="common"><!--select--></p>
</div>
<h1>...</h1>
<p class="common">...</p>

will apply to all elements of class common with a division parent

selectorA, selectorB

Selectors joined using commas are not combined:

each is a separate selector for the same rule

#everyday {
    im:shuffling;
}

something doesn't seem right here...

Luckly, the second part of CSS rules, curly braces, is much simpler:

{

rule starts here

...

}

rule ends here

The third part of CSS rules, property definitions, are actually four parts:

name

The name of the style property

:

a colon separator

value

The value of the style property

;

a semicolon to end

If you forget even 1

;

semicolon to end

the entire CSS will not work

 
 
 
 

CSS is named cascading for several reasons, but primarily because

rules cascade from parents to children

In other words: parent properties are given to children

.parent {
    propertyA:valueA;
}
.child {
    propertyA:valueB;
}
<div class="parent">
    <p><!--parent properties--></p>
</div>
<div class="parent">
    <p class="child"><!--own properties--></p>
</div>

unless a child has a rule for that property

But what if we have two rules for the same HTML element?

We always use the rule that is

most specificly selected

&

in case of a tie

farthest down the sheet

#idsUseCamelCase { ... }
.classes-use-dash-case { ... }

A naming convention is

a rule for naming certain types of things

(used to make code easier to understand)

Much like HTML, CSS has comments that are composed of three parts:

/*

opening tag

comment

comment text

*/

closing tag
