non-semantic

semantic

Semantics

are for meaningful content

Semantics is defined as

the implied meaning

of a word or phrase

Web semantics

help make sense of a web page

for both man and machine

Remember the div element

that holds anything?

Think of every element in a web page as

a sign

We use signs

to find information

Imagine if every sign said the same thing:

“this could be anything”

:(

What a depressing place

Of course, we can’t have

a sign for every detail either

(this would be very inefficient)

Let's take a middle ground and use semantics to define

general types of content

(just like we do with classes)

Here is the semantic flow:

(partial credit to HTML5 Doctor!)

Ask yourself, is this element

for navigation?

If so, use:

<nav>

(otherwise, go to next)

Ask yourself, does this element

belong at the end?

If so, use:

<footer>

(otherwise, go to next)

Ask yourself, can this element

stand alone?

If so, use:

<article>

(otherwise, go to next)

Ask yourself, is this element

secondary or unimportant?

If so, use:

<aside>

(otherwise, go to next)

Ask yourself, is this element

an example?

If so, use:

<figure>

(otherwise, go to next)

Ask yourself, is this element

multimedia?

If so, use:

<video>

or

<audio>

(otherwise, go to next)

Ask yourself, is this element

for dynamic graphics?

If so, use:

<canvas>

(otherwise, go to next)

Ask yourself, does this element

require a heading?

If so, use:

<section>

(otherwise, go to next)

Ask yourself, is this element

a heading?

If so, use:

<header>

+

<h1/2/etc>

(otherwise, go to next)

Ask yourself, is this element

a pull-quote?

If so, use:

<blockquote>

(otherwise, go to next)

Ask yourself, is this element

an address or email?

If so, use:

<address>

(otherwise, go to next)

Ask yourself, is this element

general text?

If so, use:

<p>

(otherwise, go to next)

and when none of these make sense, use

<div>

Without semantics:

<body>
    <div id="wrapper">
        <div>
            <div>...</div>
        </div>
        <div id="mainContent">
            <div id="welcome">
                <div class="intro">
                    <div>
                        <h1>Article intro header</h1>
                    </div>
                    <p>Article intro body copy</p>
                </div>
            </div>
        </div>
    </div>
</body>

With semantics:

<body>
    <div id="wrapper">
        <aside>
            <nav>...</nav>
        </aside>
        <div id="mainContent">
            <article id="welcome">
                <section class="intro">
                    <header>
                        <h1>Article intro header</h1>
                    </header>
                    <p>Article intro body copy</p>
                </section>
            </article>
        </div>
    </div>
</body>
