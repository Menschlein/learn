Lists & Tables

are high-level organizational elements

(What's his power level? Its over 9000!)

A high-level element

combines several simpler elements

to create a complex result

For example,

a list

is a container with a collection of information points

a table

is a container with lots of sub-containers

You could create a list or table

using lots of div and paragraph elements

Instead, HTML provides us with semantic elements for both

To create a list, we use either

an ordered or unordered list

<ol>...</ol>

The ordered list is denoted by ol, and is

used for sequential information

ex:

  1. List item
  2. List item
  3. List item
<ul>...</ul>

The unordered list is denoted by ul, and is

used for general information

ex:

  • List item
  • List item
  • List item

The unordered list is used most commonly and

works very well for navigation

<ul>
    <li>...</li>
</ul>

List items are denoted by li, and

are a single point of information

ol, ul {
    list-style-type: none;
}

The list’s list-style-type property defines

the appearance of a bullet point

(common values are circle, square, and none)

ex:

  • List item
  • List item
  • List item
ol, ul {
    list-style-position: outside;
}

The list’s list-style-position property defines

where the bullet point will appear

(common values are outside and inside)

By default lists display vertically, but to create a horizontal list set the

list item display property to inline

(or inline-block)

a horizontal list may look like this:

  • List item
  • List item
  • List item
  • List item
  • List item
li {
    padding: 10px;
}

The list’s padding or margin properties define

the spacing between list items

We use tables when

our data is tabular

For a long time, designers used tables to build entire websites because

tables provide easy organization

(this is now considered bad practice)

In HTML, tables are organized by

row

 

and then by

column

<table>
    
    <thead>...</thead>
    
</table>

A table header is denoted by thead and

contains column labels

<table>
    <thead>...</thead>
    
    <tbody>...</tbody>
    
</table>

A table body is denoted by tbody and

contains all table content

<table>
    <thead>...</thead>
    <tbody>
        
        <tr>...</tr>
        
    </tbody>
</table>

A table row is denoted by tr and

contains all only table cells

<table>
    <thead>...</thead>
    <tbody>
        <tr>
            
            <td>...</td>
            
        </tr>
    </tbody>
</table>

A table cell is denoted by td and

contains information

(by default cells are left aligned)

Table cells inside the table header

are denoted by th, not td

(by default header cells are center aligned and bold)

And now for your viewing pleasure, a bare-bones table:

<table>
    <thead>
        <tr>
            <th>Name</th>
            <th>Location</th>
            <th>Date</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>name1</td>
            <td>location1</td>
            <td>date1</td>
        </tr>
        <tr>
            <td>name2</td>
            <td>location2</td>
            <td>date2</td>
        </tr>
    </tbody>
</table>

<tr>

</tr>
<tr>

    <td>date2</td>

By default, tables are surrounded and sliced by borders

Thee border-width property defines

the size of the border

for example:

Name Location Date
name1 location1 date1
name2 location2 date2 
table {
    border-width: 5px;
}

Thee border-style property defines

the appearance of the border

(common values are solid, dashed, & none)

for example:

Name Location Date
name1 location1 date1
name2 location2 date2 
table {
    border-style: dashed;
}

Thee border-color property defines

the color of the border

(set in hex, rgb, & rgba)

for example:

Name Location Date
name1 location1 date1
name2 location2 date2 
table {
    border-color: #FF0000;
}
table {
    border: 1px solid black;
}

The border property is a combination of

width + style + color

By default border properties

affects all 4 sides

To set a border for a specific side

add -top, -bottom, -left, or -right

after the word border in the property name

for example:

Name Location Date
name1 location1 date1
name2 location2 date2 
table {
    border-bottom-color: #FF0000;
}

The table's border-collapse property defines

whether or not double borders can merge

(common values are collapse and separate)

for example:

Name Location Date
name1 location1 date1
name2 location2 date2 
table {
    border-collapse: collapse;
}

The border-radius property defines

the roundness of corners

(set as px, em, or %)

for example:

Name Location Date
name1 location1 date1
name2 location2 date2 
table {
    border-radius: 10px;
}

Most table styling should be

added to the table cells

To avoid confusion in large tables,

use the nth-child pseudo-selector

to add alternating background colors

div {
    background-color: #ff0000;
}

The background-color property defines

the fill color of an element

(common values are hex, rgb, rgba, and gradients)

To select every even item

use :nth-child(even)

Name Location Date
name1 location1 date1
name2 location2 date2
name3 location3 date3
name4 location4 date4

For our even rows, let's set the background-color to near white

tbody tr:nth-child(even) td {
    background-color: #dddddd;
}

To select every odd item

use :nth-child(odd)

For our odd rows, let's set the background-color to slightly darker

tbody tr:nth-child(odd) td {

}

You can also use the :hover pseudo-selector to

give a user feedback within the table

When we hover, let's set the background-color to slightly brighter
