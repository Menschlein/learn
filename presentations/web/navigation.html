Navigation

is the web

Think of the Internet as

a spider web

large enough to engulf the world

The Internet catches

billions of ideas

And is home to

millions of spiders

(we live for this information)

Wayfinding in the web defined by

links or anchors

Anchors are signs that

point to information

(they give the web structure)

Without anchors, the internet would be

an unknown space

where there is no up, down, left, or right

And in space,

no one can hear you scream

<a>...</a>

The anchor tag is denoted by 'a' and is

a link to other information

<a>Link Name</a>

Set the anchor text just like paragraph text:

between the opening and closing tags

(this is what users see as the link text)

<a href="http://www.there.com/">Link Name</a>

Set the anchor destination

using the href attribute

(this is where users will go if they click the link)

<a href="http://www.there.com/" target="_blank">Link Name</a>

Set the anchor window

using the target attribute

(_self is this window and _blank is a new window)

<a href="http://www.there.com/about.html">There: About</a>

An absolute link tells us

where to go from anywhere to there

Absolute links are like

a web page's address

(always use absolute links for external information)

<a href="about.html">About</a>

A relative link tells us

where to go from here to there

Relative links are like

getting directions

(always use relative links for navigation)

<a href="#section1">Section 1</a>
...
<div id="section1">...</div>

In-page links are anchors that point to

information in the same page

(set the href to "#" + the element's id)

In-page links are like

turning to a chapter in a book

Anchors are the

simplest interactive element

With a

click or tap

they move you across the web

What if posters had links that teleported you to other posters?

For a simple interaction we have a simple rule:

feedback for every interaction

Anchors have three notable interaction states, or

modes of being in reaction to user activity

We can control the style of these states

using CSS pseudo-class selectors

div:pseudoclass {
    ...
}

Pseudo-classes are built in selectors for

picking certain elements or element states

(they always come right after the tag or class selector)

a:hover {
    ...
}

The :hover state is triggered whenever

a mouse moves over an element

The :hover state is

valid for all elements

a:active {
    ...
}

The :active state is triggered whenever

a link is clicked or tapped

(this state ends when the click ends)

a:visited {
    ...
}

The :visited state is triggered whenever

a link has been used

By default, anchors are underlined

anchor!

to differentiate from non-interactive text

but this can be changed with the text-decoration property

Creating a

navigation bar

is essential and simple

<nav>...</nav>

Begin with a nav element...

<nav>
    <a href="index.html">Home</a>
    <a href="services.html">Services</a>
    <a href="about.html">About</a>
</nav>

...and end with one anchor element per link

The default home page for a website is always

index.html

So a URL like www.ilovegoats.com is actually

www.ilovegoats.com/index.html

Website file structure

=

device file structure

Everything after the basic URL

www.ilovegoats.com/billy/goats/gruff.html?size=little

is a folder or file on the website

Everything after the basic URL and before the last "/"

www.ilovegoats.com/billy/goats/gruff.html?size=little

is a folder

Everything after the last "/" and before the last "."

www.ilovegoats.com/billy/goats/gruff.html?size=little

is a file name

Everything after the last "." and before the first "?"

www.ilovegoats.com/billy/goats/gruff.html?size=little

is a file type

Everything after the first "?"

www.ilovegoats.com/billy/goats/gruff.html?size=little

is a query string

(don't worry about these for now)

div {
    position: static;
}

The position property defines

how an element is positioned

(common values are relative, fixed, and absolute)

By default the position property of all elements is set to position: static

which creates a continuous vertical flow

Position: relative is similar to static, but it will

move an element relative to its static position

(based on its top, bottom, left, and right properties)

example of relative positioning:

#arrow2 {
    position: relative;
    left: 100px;
}

Position: absolute will take an element out of the vertical flow and

1. find the closest parent with position: relative

2. move it relative to that parent's position

(but only for the properties you define)

3. retain its static position values

(for the properties you did not define)

example of absolute positioning:

#arrow2 {
    position: absolute;
    left: 100px;
}

(notice that the arrow is still at its original top value)

Position: fixed will take an element out of the vertical flow and

move it relative to a user’s browser window

(based on its top, bottom, left, and right properties)

Use fixed positioning to create sticky elements that

always stay in view

(this is super useful when used with a navigation bar)

.navbar-fixed {
    box-shadow: 0 1px 10px rgba(0, 0, 0, .1); 
}

When using sticky elements, improve the contrast by using the box-shadow property to

give the element a drop shadow

It is also common practice for a web page to have a footer

Web footers usually contain:

copyrights,

notices,

sponsors,

& site maps

A site map is

a listing of all of a website's pages

(usually organized into lists of anchors)

A site map is great for

simplifying page navigation

but the trade-off is that they often

take up a lot of space

(so we don't use them at the top)
