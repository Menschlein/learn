However, checking for mobile device or browser type

is fools-gold

Technology changes so quickly that any device or browser could suddenly change

Checking for feature support is much more stable

To reliably check for feature support on the web

use CSS @media queries

/* basic rules */
body {
    ...
}

/* rules for non-basic situations */
@media state1 and state2 and... {
    body {
        ...
    }
}

A @media query defines

a page state + state rules

A @media query is basically

a simple logic statement

The browser asks:

@media state1

For this @media query, is the page state = to the @media state?

@media state1

If yes, apply the @media query's rules

@media state1

If no, ignore the @media query's rules

But how do we detect desktop vs mobile without detecting the device itself?

Desktop devices have

large screen sizes

(between 768px and 1600px width)

Tablet devices have

medium screen sizes

(up to 768px width in portrait mode)

Phone devices have

small screen sizes

(up to 480px maximum width in landscape mode)

A @media query can detect screen sizes using the

state phrases “min-” and “max-”

@media (min-width:1200px) {
    /* rules for large desktops */
}

The min- statetakes the form of

(min-dimension:size)

(where dimension can be either width or height)

@media (max-width:480px) {
    /* rules for phones */
}

The max- statetakes the form of

(max-dimension:size)

(where dimension can be either width or height)

For example, use

@media (max-width:480px)

to detect a phone

For example, use

@media (max-width:767px)

to detect a tablet

For example, use

@media (min-width:768px) and (max-width:979px)

to detect a small desktop

For example, use

@media (min-width:1200px)

to detect a large desktop

Remember, small screen sizes work much better

with larger sizes and vertical layouts

For phones and small tablets, consider changing the page into

a vertical stack with 20 px font-size

To change a row/column grid into a vertical stack:

1. Set columns float properties to none

2. Set columns display to block

3. Set columns width to auto or 100%

4. Set row & column margins to 0

100% width = 1x row = 12x columns + 11x gutters

each gutter = 1/4 of column width

100% width = 1x row = 12x columns + 11x 1/4 columns

each column = 100 % width / 14.75 columns = ~6.78%

100% width = 1x row = 12x columns (6.78%) + 11x gutters (1.69%)

In the case that a page must retain the grid, use a fluid grid with

% widths instead of px widths

(for universal support across screen sizes)
